# 《三國單騎傳》v0.24.0-pre 階段性評估報告

## 1. 評估摘要

《三國單騎傳》目前已完成第一章可完整遊玩的垂直切片：首頁、開場動畫、四位角色、8 關戰鬥、路線事件、Boss 特性、勝敗結果、分享、PWA 與部署流程都已具備。整體已不再只是原型，而是接近可對外試玩的作品。第一章主要圖片資產已補齊，TTS 已有 52 筆 ready，BGM 與卡牌音效也完成第一版，因此文、圖、聲、影的整合度明顯高於一般早期 Demo。手機 UX 已經有底部手牌操作區、狀態 badge、敗退標記與選擇高亮，方向正確。

目前最大問題不是功能缺口，而是「外部玩家第一次玩是否夠順」與「普通模式是否太容易」。`npm run balance` 顯示 200 局整體勝率 98.5%，四位角色勝率差距只有 2%，代表平衡差異不大，但挑戰性偏低。資源面也需要注意：目前 `public/audio`、`public/images`、`public/videos` 合計約 123.87 MB，其中 PNG 約 86.76 MB、開場 MP4 約 24.16 MB，手機網路首次載入可能有壓力。工程面測試數量充足，22 個測試檔、241 個測試通過，但 visual manifest 與 gap manifest 的 status 表達不完全一致，後續維護時容易混淆。

本報告建議下一階段先做 v0.24.0「完整體驗 QA 修正版」，不要急著開第二章或大改戰鬥。先補上手機真機 QA、資源體積控管、普通 / 挑戰模式規劃、聲音混音檢查與首頁到通關的端到端檢查，再進入遊戲性拓展會更穩。

## 2.5 分面向審查

### A. 玩家第一印象

優點：首頁已移除版本特色與修改記錄，主流程集中在開場動畫、選擇武將與開始遊戲。首頁能透過「選擇武將，闖過 8 關，挑戰呂布」快速說明核心玩法。李詩民｜AI 架構師以彩蛋角色方式出現，和「AI 協作開發｜李詩民」的作品定位有呼應。

問題：分享、安裝、LINE 提醒、BGM 控制都放在首頁，雖然必要，但對第一次進入的玩家仍可能形成入口前的資訊噪音。開場動畫若自動被視覺上放得太前，玩家可能還不知道遊戲規則就先看影片。

建議優化：保留目前精簡首頁，不要重新加入版本展示。下一步應檢查首頁首屏在手機 390px 寬度下是否一眼看到「選武將」與「開始遊戲」。PWA / LINE 提醒維持非阻擋式，並在關閉後不要反覆出現。

### B. 手機 UX

優點：底部固定手牌區已整合主要按鈕，response phase 的「使用閃 / 承受傷害 / 以斬作閃」更接近手牌區。戰鬥 HUD 已顯示 HP、狀態 badge、裝備 badge、Boss 特性與敗退標記。reward / event / route / routeEvent / observation 的選擇卡有 glow 與 pulse，點擊意圖比早期清楚。

問題：在四位角色、三件裝備、低血量、Boss 狀態與傷害提示同時出現時，手機 HUD 仍可能資訊過載。底部固定區需要真機檢查 safe area，特別是 iPhone Safari 的底部工具列與 PWA standalone 模式。若圖片載入慢，routeEvent 與 stage 背景可能造成短暫視覺空白。

建議優化：下一版建立手機 QA 清單，至少測 iPhone Safari、Android Chrome、LINE 內建瀏覽器與 PWA standalone。用截圖記錄 player phase、defense phase、reward、routeEvent、Boss defeat 五種狀態，確認不需要頻繁上下滑動。

### C. 戰鬥節奏與遊戲性

優點：第一章 8 關長度適合展示完整 loop，卡牌種類涵蓋攻擊、防禦、回血、抽牌、破防、裝備與戰術。裝備 badge 已讓斬 +1、首攻閃避、兵書 +1 抽牌更直覺。路線事件提供補給、策略與風險，Boss 呂布有無雙壓迫與戰神回血，不再只是高血量敵人。

問題：模擬勝率過高，導致 Boss 壓迫可能只在視覺與敘事上成立，數值挑戰不足。路線事件雖豐富，但模擬策略高度偏向險道，山道幾乎沒有被選。若玩家連續勝利，8 關可能偏像展示流程而不是 Roguelike 挑戰。

建議優化：不要直接把普通模式調難，先新增挑戰模式或通關後解鎖挑戰規則。普通模式維持教學展示，挑戰模式再加強第 7～8 關、Boss 第二階段或路線代價。

### D. 平衡性

優點：四位角色勝率差距只有 2%，李詩民加入後沒有明顯拉開角色差距。關羽、趙雲、諸葛亮、李詩民的技能差異仍清楚：爆發、攻防轉換、控牌、策略調度。

問題：整體勝率 98.5% 過高，李詩民 100% 需要觀察是否因抽牌與路線事件回血過於穩定。死亡集中在第 8 關 2 次與第 3 關 1 次，代表大部分流程缺少真正失敗壓力。路線選擇分佈顯示險道 946、官道 447、山道 2，山道定位在模擬中沒有被充分使用。

建議優化：下一輪 balance 應先把模式分開：普通模式用於試玩與教學，挑戰模式用於玩家重玩。再針對挑戰模式調整呂布、張梁、張寶、險道代價與高報酬事件。

### E. 文、圖、聲、影完整度

優點：劇情、台詞、旁白、敵人登場、敵人擊敗、路線事件與勝敗旁白已完整串接。圖像方面第一章主要資產已補齊。聲音方面已有 TTS 52 筆 ready、卡牌音效、擊敗 cue 與 3 首 BGM。開場動畫提供作品第一印象，整體整合度高。

問題：關羽、趙雲、諸葛亮仍有補充戰鬥語音 planned，角色語音完整度不如李詩民。BGM、TTS、SFX 同時開啟時尚未有 ducking 或音量基準，可能互相蓋聲。資源總量偏大，尤其 PNG 與 MP4 對手機首次載入有風險。

建議優化：先建立聲音混音規則與一鍵靜音，再補齊原三位武將戰鬥語音。圖片與影片應新增 size report，不要等資源繼續增加才處理。

### F. PWA / 分享 / 安裝體驗

優點：manifest、icons、metadata、ShareGameButton、InstallPrompt、LINE 內建瀏覽器提醒與 standalone 判斷都已建立。分享入口放在首頁、遊戲更多操作與結果頁，不干擾主要出牌區。LINE 提醒有複製連結與平台教學。

問題：PWA 安裝在 iOS Safari、Android Chrome、LINE 外部瀏覽器之間差異大，僅靠程式碼測試仍不足。安裝提示與分享提示若同時出現，可能在首頁造成訊息密度偏高。

建議優化：以真機 QA 檢查 PWA 安裝名稱、icon、standalone 隱藏提示、LINE 轉外部瀏覽器。若要對外發表，可新增一張簡短「加入主畫面」教學圖，不要再加長首頁。

### G. 工程穩定性

優點：測試覆蓋面廣，包含 engine、manifest、PWA、browser environment、BGM、TTS、圖片、手機 UX、裝備顯示、李詩民技能與 balance simulator。audio、voice、BGM、install prompt 多數 helper 皆 SSR-safe，build 目前穩定。

問題：manifest 類型分散，TTS、visual、image gap、BGM、card sound 各自維護，長期可能產生 status 與路徑不一致。`npm audit` 有 Next 依賴鏈的 PostCSS moderate 警告，但 force fix 會降版，不可自動處理。尚未有 asset size report 或 dead asset 檢查。

建議優化：下一版新增資產檢查工具，列出 public 最大檔案與 manifest path 是否存在。visual manifest 的 status 應統一，避免 ready 資產沒有明確標示。

## 3. 優點

1. 第一章已形成完整閉環：選角、開場、戰鬥、路線、Boss、勝敗結果都可串起。
2. 首頁已精簡為遊戲入口，不再像版本展示頁。
3. 四位角色定位清楚，李詩民彩蛋角色自然連到 AI 協作開發主題。
4. 手機底部操作區把手牌與主要按鈕集中，符合直屏遊玩方向。
5. 敵人敗退提示、灰階與 stamp 強化了戰鬥完成感。
6. 選擇階段有高亮與提示，降低玩家不知道該點哪裡的風險。
7. 第一章圖像資產完整，視覺呈現已可支撐對外試玩。

10. 測試覆蓋資料、helper、manifest、PWA、BGM、戰鬥狀態、李詩民技能與平衡模擬，維護基礎穩。

## 4. 主要問題

### P0：立即影響遊玩或理解

- 普通模式挑戰性偏低：200 局模擬整體勝率 98.5%，李詩民 100%，其他三位也達 98%。
- 手機首次載入資源可能偏重：圖片約 86.76 MB、影片約 24.16 MB、MP3 約 12.95 MB，對 LINE / 行動網路使用者有壓力。
- 對外試玩前缺少一輪真機 QA 清單：PWA 安裝、LINE 外部瀏覽器、BGM 啟動、語音播放、橫向 / 直向切換都需要實機確認。

### P1：影響體驗但不阻塞

- 路線選擇分佈偏斜：basic-safe-strategy 1395 次路線中，險道 946 次、官道 447 次、山道 2 次，山道生存定位在模擬策略下幾乎不被選。
- 呂布 Boss 具備特性，但高勝率表示尾王壓迫仍可能不足。
- TTS 還有 15 筆 planned，多為關羽、趙雲、諸葛亮補充戰鬥語音；李詩民完整度反而高於原三位。
- 音效、語音、BGM 已分開控制，但尚未建立音量預設建議與混音準則。
- 開場動畫與首頁 BGM 的交互已處理，但 24 MB 影片仍可能讓首次體驗等待偏長。
- 手機 HUD 已壓縮，但四位角色、裝備、Boss badge、狀態 badge 同時出現時仍可能資訊過密。
- result page 有分享入口，但缺少更明確的「再玩不同角色」誘導。
- visual manifest 的 `status` 欄位為 optional，與 image gap / ready manifest 的 status 表達不一致。
- `npm audit` 有 2 個 moderate 警告，修復建議會造成 Next 重大降版，需追蹤而非直接修。

### P2：後續優化

- 尚未有正式作品介紹頁或對外發表素材。
- 沒有玩家行為統計，無法知道真實玩家卡在哪一關或是否關閉聲音。
- 沒有 asset size report，自動化檢查資源增長。
- 尚未建立完整無障礙檢查，例如鍵盤操作、aria label 與色彩對比掃描。
- 第二章、更多卡牌、Boss 第二階段與挑戰模式仍在後續拓展階段。

## 6. 下一階段建議路線

### A. 玩家體驗打磨線

目標：把目前已完成的第一章打磨到適合外部玩家直接試玩。

適合原因：目前功能量已足夠，最大風險在手機真機、載入、聲音、提示與 QA，而不是內容不足。

版本步驟：

1. v0.24.0｜完整體驗 QA 修正版：建立 QA checklist、修手機真機問題、確認 PWA / LINE / BGM / TTS。
2. v0.24.1｜資源體積優化版：圖片與影片壓縮、延後載入策略、asset size report。
3. v0.24.2｜聲音混音與一鍵靜音版：建立 BGM / SFX / TTS 音量基準與快速靜音。
4. v0.24.3｜結果頁與再玩引導版：強化通關後分享、換角色、挑戰模式入口。

### B. 遊戲性拓展線

目標：提高可重玩性與挑戰深度。

適合原因：目前基礎系統已穩，但 simulator 顯示普通模式偏容易，拓展應透過模式分層而不是直接加難。

版本步驟：

1. v0.24.0｜普通 / 挑戰模式版：保留普通模式，新增挑戰模式規則。
2. v0.24.1｜呂布第二階段版：挑戰模式下加入 Boss 第二階段行動或額外特性。
3. v0.24.2｜稀有卡與角色專屬獎勵版：提高構築變化。
4. v0.25.0-pre｜第二章規劃版：先出文件與資料 manifest，不急著實作。

### C. 對外發表線

目標：把遊戲整理成可展示的 AI 協作開發作品。

適合原因：專案已具備 PWA、分享、圖像、語音、BGM 與部署流程，適合做作品案例。

版本步驟：

## 7. 建議的下一個版本

建議下一個版本做：

**v0.24.0｜完整體驗 QA 修正版**

原因：目前第一章內容、圖像、語音、BGM、PWA 與分享都已達到可試玩門檻，下一步最有價值的是把整條玩家路徑跑順，而不是立刻做第二章或加難。若直接調高數值，可能破壞普通玩家的第一輪體驗；若先做挑戰模式，又可能忽略手機載入、LINE、PWA、混音與結果頁這些對外試玩更直接的問題。因此 v0.24.0 應先建立 QA checklist、資源體積報告、手機真機修正與聲音混音基準。之後再以 v0.24.1 或 v0.25.0 進入挑戰模式或內容拓展。

## 9. 結論

目前版本已適合進入「小範圍外部試玩」前的最後 QA 階段，但不建議立刻大規模公開或直接開第二章。第一章的內容完整度已高，文、圖、聲、影與 PWA 整合也足以展示作品價值；真正需要補的是手機真機體驗、資源載入、聲音混音、平衡難度分層與對外試玩流程。下一步最合理方向是 v0.24.0｜完整體驗 QA 修正版，先把第一章打磨成穩定、清楚、手機上順手的試玩版本，再決定是否接挑戰模式、第二章或對外展示整理。
